# Domain - Trust and Reputation

## Principio

La confianza debe ser comprensible. No mostrar porcentajes ni puntajes opacos si
el usuario no puede entender de donde salen.

## Informacion visible recomendada

- identidad confirmada;
- calificacion general;
- cantidad de operaciones o trabajos realizados;
- antiguedad;
- una o dos cualidades resumidas.

Ejemplo:

```text
Identidad confirmada
4,8 de 5
42 operaciones realizadas
Cumple y responde rapido
```

## Informacion interna posible

- telefono confirmado;
- identidad confirmada;
- operaciones completadas;
- cancelaciones;
- reclamos;
- rapidez de respuesta;
- cumplimiento;
- actividad sospechosa;
- consistencia de identidad;
- documentacion presentada;
- revisiones administrativas.

## Validacion progresiva

La validacion debe depender de riesgo, categoria, valor, frecuencia y funciones
solicitadas. No debe bloquear el uso inicial de bajo riesgo.

Estados sugeridos:

- none;
- phone_confirmed;
- identity_confirmed;
- documentation_presented;
- documentation_reviewed;
- suspended;
- revoked.
